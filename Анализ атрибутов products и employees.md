Атомарность на уровне ячейки таблицы означает, что значение в ней должно быть единственным.

**1\. Сущность products** 

1. product\_id — каждому товару присваивается только один уникальный номер. Не может быть у одного товара несколько ID.   
2. product\_name — ячейка содержит только одно конкретное название, то есть в этой ячейке нет других названий.  
3. price — это значение показывает одну конкретную актуальную.  
4. category\_id — товар принадлежит только одной категории, под конкретным ID.  
5. class — содержит только один код класса.  
6. modify\_timestamp — отображает один конкретный момент времени последнего обновления.  
7. resistant —  не может быть одновременно  двух противоречивых состояний (Yes/No).  
8. is\_allergic — не может быть одновременно  двух противоречивых состояний (Yes/No).  
9. vitality\_days —  конкретное число дней, тут не может быть списка периодов в рамках одной ячейки.

## **2\. Сущность employees**

1. employee\_id — каждому товару присваивается только один уникальный номер.   
2. first\_name — внутри каждой ячейки находится ровно одно неделимое строковое значение.  
3. middle\_initial — внутри каждой ячейки находится ровно одно неделимое строковое значение.  
4. last\_name — внутри каждой ячейки находится ровно одно неделимое строковое значение.  
5. birth\_date — человек рождается один раз. Ячейка хранит одну конкретную дату  
6. gender — хранит строго один символ, который определяет пол сотрудника.  
7. city\_id — конкретный один город.  
8. shop\_id — в данный момент сотрудник закреплен за одним конкретным номером магазина.  
9. hire\_date — человек принимается на определенную должность в определенный день. Ячейка хранит одну конкретную дату
